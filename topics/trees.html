1. עצי החלטה לא-פרמטרי

  • מודל לא-לינארי ולא-פרמטרי: לא מניח צורת קשר; מספר הפרמטרים (פיצולים) גדל עם הנתונים.
  • מחלק את מרחב התכונות למלבנים מקבילי-צירים — כל פיצול הוא חיתוך ניצב לציר.
  • שקוף ומובן, עובד עם נתונים קטגוריאליים ונומריים.
  • מציאת העץ האופטימלי היא בעיה NP-קשה — לכן משתמשים בחיפוש חמדני Top-Down.
חסרונות נוטים ל-Overfitting (עץ עמוק שמשנן רעש), שונות גבוהה (שינוי קטן ⟵ מבנה שונה), וקושי עם הפרדות אלכסוניות (למשל $y=x$ דורש מדרגות רבות).

2. אנטרופיה ו-ID3 חמדני

אנטרופיה
מודדת אי-ודאות (חוסר טוהר): $\;H(S)=-\sum_i p_i\log_2 p_i$. נמוכה = טהור; גבוהה = מבולגן (למשל 50%-50% ⟵ $H=1$).
Information Gain
$$IG(S,A)=H(S)-\sum_{v\in\text{Values}(A)}\frac{|S_v|}{|S|}H(S_v).$$ ID3 בוחר בכל צומת את התכונה $A$ שממקסמת את ה-IG (מקטינה הכי הרבה את אי-הוודאות).
נתון: 14 דגימות: 9 חיוביות, 5 שליליות. תכונה $A$ מפצלת ל: קבוצה 1 (6+, 2−) וקבוצה 2 (3+, 3−).
אנטרופיה מקורית: $H(S)=-\tfrac9{14}\log_2\tfrac9{14}-\tfrac5{14}\log_2\tfrac5{14}=0.410+0.530=0.940$.
קבוצה 1 (8 דגימות, 6+2−): $H=-\tfrac68\log_2\tfrac68-\tfrac28\log_2\tfrac28=0.311+0.5=0.811$.
קבוצה 2 (6 דגימות, 3+3−): $H=1$ (מפוצל שווה).
אנטרופיה משוקללת: $\tfrac8{14}(0.811)+\tfrac6{14}(1)=0.464+0.429=0.892$.
Information Gain: $IG=0.940-0.892=0.048$. ככל שגבוה יותר — הפיצול טוב יותר.
# ID3
BuildTree(Data, Features):
  if all + : return leaf(+)
  if all − : return leaf(−)
  if no features: return leaf(majority)
  A = argmax IG(Data, A)
  for each value v of A:
    subtree = BuildTree(Data[A=v], Features \ {A})

3. CART, Gini וגיזום CART

מדד Gini
$Gini = 1-\sum_{i=1}^c p_i^2$. אם כל הדגימות מאותה מחלקה ⟵ $Gini=0$; מקסימום $\tfrac12$ (בינארי). מהיר יותר מאנטרופיה (אין לוגריתם). CART בונה עץ בינארי ותומך גם במשתנים נומריים.
נתון: צומת עם 4 חיוביות ו-6 שליליות.
Gini: $1-(0.4^2+0.6^2)=1-(0.16+0.36)=1-0.52=0.48$ — טוהר נמוך (קרוב ל-0.5).
אם היו 9 חיוביות ו-1 שלילית: $1-(0.81+0.01)=0.18$ — טהור יותר.
גיזום (Pruning)
Pre-pruning: עוצרים מראש (הגבלת עומק, מינימום דגימות, סף IG). יעיל אך עלול לעצור מוקדם מדי. Post-pruning: נותנים לעץ לגדול ואז חותכים ענפים שלא פוגעים בדיוק על סט ולידציה — טוב יותר אך יקר יותר.

4. Boosting ו-XGBoost אנסמבל

Weak Learner: מודל טוב מעט יותר מניחוש אקראי (>50%). Boosting משלב הרבה לומדים חלשים סדרתית ללומד חזק: $\;\text{Strong}=\sum_{m=1}^M \gamma_m h_m(x)$. כל לומד חדש מתקן את שאריות הקודם.

Gradient Boosting
במקום לחזות את היעד מאפס, כל עץ חדש חוזה את השאריות $r_i=y_i-f_{old}(x_i)$: מאמנים $h_m=\arg\min_h\sum_i(r_i-h(x_i))^2$, ומעדכנים $f_{new}=f_{old}+\alpha h_m$. זהו Gradient Descent במרחב הפונקציות (עבור MSE, השארית היא בדיוק הגרדיאנט השלילי).
נתון: מחירים אמיתיים $y=[100,120,150]$; מודל נוכחי ניבא $f_{old}=[90,130,140]$.
שלב 1 — שאריות: $r=y-f_{old}=[+10,-10,+10]$.
שלב 2: מאמנים עץ חדש $h$ לחזות את $[10,-10,10]$ (ולא את $y$ המקורי!).
שלב 3 — עדכון (α=1): אם $h$ ניבא ~$[10,-10,10]$, אז $f_{new}=f_{old}+h\approx[100,120,150]$ — קרוב ליעד. ✓
AdaBoost
לעומת GB, ה-AdaBoost פועל בסיווג: מגדיל את משקל הדגימות שסווגו לא-נכון, נותן לכל לומד משקל $\alpha_m$ לפי דיוקו, ומשתמש ב-Stumps (עצים בעומק 1). התחזית: $H(x)=\mathrm{sign}\big(\sum_m\alpha_m h_m(x)\big)$.
